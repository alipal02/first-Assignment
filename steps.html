Backend API Development: Step-by-Step Guide

Step 1: Planning and Architecture

I reviewed all the strict assignment constraints, specifically the rule to utilize zero external frameworks such as Express.js or Axios. I planned out the need for two core structures: a central router (server.js) and an endpoint handler module (api.js).

Step 2: Core Server Setup (server.js)

Created a native HTTP server listening on port 3000. It reads incoming connection streams, automatically rejects requests dealing with foreign endpoints, and strictly forwards all traffic beginning with the /movies path into the centralized API handler module.

Step 3: Database Integrations (api.js)

I utilized the built-in fs and path modules to securely wrap the designated data source (movies-db.json) behind two helper promise-based functions: readDB() and writeDB(). This guarantees consistent synchronous reads and writes while preventing data corruption.

Step 4: Request Parsing & Validation

I constructed a standard lightweight body-parser to sequentially listen to stream 'data' and 'end' events. Before mutating the dataset structure, the server consistently invokes strict conditionals verifying title integrity alongside enforcing explicit numeric typing rules for both year and rating fields.

Step 5: Implementing Standard Endpoints

I manually constructed explicit regex string parameters to capture wildcard resource queries like /movies/:id seamlessly. Leveraging native array properties, I developed logic to find, findIndex, splice, and push object modifications resolving all GET, single GET, POST, PATCH, and DELETE methods efficiently reflecting instantly inside the JSON manifest.

Step 6: Delivering Bonus Requirements

Iterating above standard demands, I attached Node's core url parsing module enabling programmatic isolation of query arguments facilitating instantaneous array slicing filtering through destructured ?search and ?limit instructions just before transmission rendering.

Step 7: Robust Error Handling & HTTP Context Export

I embedded the logic processing into a sweeping overarching Try-Catch container purposefully crafted to intercept syntactical violations universally translating inner exceptions to pristine 400 Bad Request triggers or resilient 500 error outputs strictly conforming to RESTful paradigms.
